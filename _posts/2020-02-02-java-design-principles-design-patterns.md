Today I've realized that I've been working as software developer/engineer for almost 10 years. 
It's been such a great experience so far mainly because I've been evolving more and more as human being
when dealing with amazing people that I met on this long journey. 

Other than that, my point on this post is to share a great document written by [Robert C. Martin](https://en.wikipedia.org/wiki/Robert_C._Martin). 
Even though this document was written in 2000, I'm really surprised how it is still represents most of the system design I've worked so far after 20 years.
This document could be represented as the parent of the [SOLID](https://en.wikipedia.org/wiki/SOLID) principles.

I think we, as developers, could pick a free time to read it and reflect about it during the phase we're buildinig something new, although
the technical decisions aren't always the problem.
